### 4.1 CAPL 코딩 표준

**참고 문서**: [CONTRIBUTING.md](file:///Users/juns/code/work/mobis/PBL/CONTRIBUTING.md)

#### 파일 구조
```capl
/*
 * Module: Ambient Lighting Control
 * Description: RGB ambient lighting zone management
 * Author: [이름]
 * Date: 2026-02-05
 * Version: 1.0.0
 * ASIL: QM
 */

// Includes
#include "ambient_interface.cin"

// Constants
const int ZONE_COUNT = 8;
const int MAX_BRIGHTNESS = 100;

// Global variables
int gCurrentBrightness = 50;

// Function declarations
void InitializeAmbientLighting();
void SetZoneColor(int zone, byte r, byte g, byte b);

// Event handlers
on start {
    InitializeAmbientLighting();
}

// Function implementations
void InitializeAmbientLighting() {
    write("Initializing ambient lighting...");
    // 구현
}
```

#### 명명 규칙
- **전역 변수**: `g` 접두사 + CamelCase
  - 예: `gAmbientBrightness`, `gColorMode`
- **상수**: 대문자 + 언더스코어
  - 예: `MAX_BRIGHTNESS`, `ZONE_COUNT`
- **함수**: CamelCase, 동사로 시작
  - 예: `SetAmbientColor()`, `GetBrightness()`

```

3. 고장 감지 대기 (2초)
4. DTC P0C00 설정 확인
5. UDS 서비스 0x19로 DTC 읽기
6. 고장 램프 점등 확인
7. 고장 해제 및 복구 확인
```

**예상 DTC**: `P0C00` - Ambient Light Sensor Circuit

#### 시나리오 2: OTA 업데이트 무결성 고장
```
1. OTA 업데이트 시작
2. 업데이트 패키지에 체크섬 오류 주입
3. 업데이트 거부 확인
4. DTC P0600 설정 확인
5. 이전 소프트웨어 버전 유지 확인
```

**예상 DTC**: `P0600` - Software Verification Failure

### Q2: 요구사항 ID는 어떻게 부여하나요?
**A**: 형식: `[도메인]-[타입]-[번호]`
- 도메인: `LGT` (조명), `DIAG` (진단), `IVI` (IVI)
- 타입: `AMB` (앰비언트), `DASH` (대시보드), `OTA`, `UDS` 등
- 번호: 001부터 시작

예시: `LGT-AMB-001`, `DIAG-OTA-015`

### Q3: ASIL 레벨은 어떻게 결정하나요?
**A**: [safety-management/README.md](file:///Users/juns/code/work/mobis/PBL/safety-management/README.md) 참조
- QM: 안전 영향 없음 (예: HVAC)
- ASIL A: 낮은 안전 영향 (예: Window, Dashboard Lighting)
- ASIL B: 중간 안전 영향 (예: ADAS Lighting, Diagnostics, OTA)
