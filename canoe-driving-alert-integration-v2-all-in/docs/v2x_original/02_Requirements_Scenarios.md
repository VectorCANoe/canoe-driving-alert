# 요구사항 및 시나리오

**Date**: 2026-02-24

## 요구사항

### 공통 인프라 — 모든 시나리오의 기반

| Req. ID | 요약 | 설명 |
|---------|------|------|
| Req_V01 | 다중 차량 노드 구성 | 경찰차, 일반차(3대), 선두 군용차, 후행차(3대), 서버, 물류차를 독립된 차량 노드로 구성한다 |
| Req_V02 | Ethernet V2V 채널 | 차량 간 통신은 Ethernet 1개 버스로 구성한다. 1대가 메시지를 보내면 연결된 모든 차량이 동시에 수신할 수 있다 |
| Req_V03 | CAN 내부 제어 채널 | Ethernet으로 받은 명령은 차량 내부 CAN을 통해 계기판, 앰비언트 라이트, 사이렌 등 각 장치에 전달된다 |

---

### 시나리오 1 — 경찰차 긴급 통행권

| Req. ID | 요약 | 설명 |
|---------|------|------|
| Req_E01 | 긴급 모드 전환 | 경찰차 화면(Panel)의 버튼을 누르면 긴급 출동 상태(Emergency = 1)로 즉시 전환된다 |
| Req_E02 | Ethernet 긴급 메시지 전송 | 긴급 출동 상태가 되면 Ethernet으로 연결된 모든 차량에게 긴급 메시지(차량 ID 포함)를 즉시 발송한다 |
| Req_E03 | 일반차 계기판 강제 경고 | 긴급 메시지를 수신한 일반차는 계기판에 "긴급 차량 접근 중 — 우측 양보" 팝업을 표시하고 앰비언트 라이트를 주황색으로 점멸시킨다 |
| Req_E04 | 긴급 모드 해제 | 경찰차가 긴급 모드를 해제하면 모든 일반차의 경고가 자동으로 사라진다 |

---

### 시나리오 2 — 군집 차량 위협 대응

| Req. ID | 요약 | 설명 |
|---------|------|------|
| Req_M01 | 위협 단계 입력 | 선두 군용차에서 위협 수준을 3단계(1: 경계 / 2: 회피 기동 / 3: 스텔스)로 설정할 수 있다 |
| Req_M02 | 군집 동시 명령 전송 | 선두차가 위협 단계를 설정하면 Ethernet으로 모든 후행차에 동일한 명령을 동시에 전송한다 |
| Req_M03 | 후행차 단계별 실행 | 명령 수신 즉시 실행한다. Level 1 → 경계등 점멸 / Level 2 → 비상 제동 시뮬레이션 / Level 3 → 전체 조명 소등(스텔스 모드) |
| Req_M04 | 위협 해제 및 복귀 | 선두차가 해제 명령을 보내면 모든 후행차가 정상 상태로 복귀한다 |

---

### 시나리오 3 — 물류차 임무 전환 OTA

| Req. ID | 요약 | 설명 |
|---------|------|------|
| Req_P01 | 국가 서버 OTA 명령 | 국가 명령 서버(Server_Node) 화면에서 재난 코드를 입력하면 물류차에 긴급 업데이트 명령을 보낸다 |
| Req_P02 | Ethernet OTA 전송 | 서버가 Ethernet을 통해 "긴급차량 모드" 펌웨어를 물류차에 전송한다. 전송 진행률이 화면에 표시된다 |
| Req_P03 | 무결성 검증 | 전송 완료 후 데이터 오류 여부를 자동으로 확인(CRC 검증)한다. 오류 시 이전 상태로 자동 복구한다 |
| Req_P04 | 임무 전환 완료 | 검증 통과 후 차량이 재시작된다. 재시작 후: 사이렌 ON + 앰비언트 적청 교차 + 속도 제한 해제 + 계기판 "긴급차량 모드" 표시 |

## 시나리오 테스트

### 시나리오 1 — 경찰차 긴급 출동

> 전제: 경찰차 1대 + 일반차 3대가 Ethernet으로 연결된 상태

| Scene | 상황 | 확인 항목 |
|-------|------|---------|
| **Scene.E1** | 경찰차 Panel에서 "긴급 출동" 버튼을 누른다 | 경찰차 긴급 상태 활성화 확인 |
| **Scene.E2** | 일반차 A, B, C의 계기판을 확인한다 | 3대 모두 동시에 "긴급 차량 접근 중, 우측 양보" 팝업 + 앰비언트 주황 점멸 |
| **Scene.E3** | "긴급 출동" 버튼을 다시 눌러 해제한다 | 3대 모두 경고 자동 소거 확인 |

---

### 시나리오 2 — 군집 위협 대응

> 전제: 선두차 1대 + 후행차 3대가 Ethernet으로 연결된 상태

| Scene | 상황 | 확인 항목 |
|-------|------|---------|
| **Scene.M1** | 선두차에서 "위협 Level 1 (경계)" 명령을 보낸다 | 후행차 1/2/3 경계등 동시 점멸 확인 |
| **Scene.M2** | "위협 Level 2 (회피 기동)" 으로 단계를 높인다 | 후행차 전체 비상 제동 시뮬레이션 동시 실행 확인 |
| **Scene.M3** | "위협 Level 3 (스텔스)" 명령을 내린다 | 후행차 전체 조명 즉시 소등 동기화 확인 |
| **Scene.M4** | 선두차에서 "위협 해제" 명령을 보낸다 | 후행차 전체 정상 상태 복귀 확인 |

---

### 시나리오 3 — 물류차 임무 전환

> 전제: 국가 서버 1대 + 물류차 1대가 Ethernet으로 연결된 상태

| Scene | 상황 | 확인 항목 |
|-------|------|---------|
| **Scene.P1** | 서버 Panel에서 재난 코드를 입력하고 OTA를 시작한다 | 물류차 화면에 "긴급 업데이트 수신 중" 표시 확인 |
| **Scene.P2** | 펌웨어 전송이 진행된다 | 진행률 0%→100% 표시 + CRC 검증 통과 + 차량 재시작 확인 |
| **Scene.P3** | 재시작 후 긴급차량 모드를 확인한다 | 사이렌 ON + 앰비언트 적청 교차 + 계기판 "긴급차량 모드" 확인 |
| **Scene.P4** | 전송 중 오류를 발생시킨다 | 자동으로 이전 물류차 상태로 복구 + "업데이트 실패" 표시 확인 |
