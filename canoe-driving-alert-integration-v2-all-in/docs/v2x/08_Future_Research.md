# 향후 발전 방향 및 연구 제언

> 특수목적 차량 V2V 통신 검증 플랫폼 — 확장 가능성

---

## 1. 현재 구현의 의의 및 한계 (PoC)

본 프로젝트는 CANoe 시뮬레이션 환경에서 CAN과 Ethernet(UDP 브로드캐스트)을 융합하여 차량 간 협력 제어의 핵심 로직을 검증했습니다.

### 1.1. 실제 V2X 환경 전환(Porting)을 위한 기술적 제언
본 구현은 논리적 정합성 검증에 최적화되어 있으며, 향후 실제 양산 V2X(WAVE, C-V2X) 네트워크망으로 확장하기 위해서는 물리적 조건에 대한 선제적 고려가 필요합니다.

*   **A. 무선 통신(RF) 채널의 신뢰성 극복**: 현재의 Ethernet 기반 시뮬레이션은 이상적인 네트워크망을 가정합니다. 향후 실도로 물리 계층(PHY) 적용 시에는 실제 무선 환경에서 필연적으로 발생하는 **전파 간섭(Interference), 패킷 손실, 지연 스파이크**에 대응하기 위한 통신 보정 및 예외 처리 알고리즘 추가가 요구됩니다.
*   **B. 국제 표준 프로토콜 스택 적용**: 본 PoC는 로직 검증 효율을 높이고자 자체 정의된 경량화 UDP 페이로드를 사용했습니다. 이를 상용 V2X망에 통합하기 위해서는 GeoNetworking 캡슐화, 보안 인증(Security Header), 그리고 ASN.1 구조체 파싱 모듈 등 **표준 통신 규격 기반의 인코딩/디코딩 스택** 결합이 필요합니다.

| 구분 | 현재 구현 (PoC) | 실제 V2X 환경 |
|------|----------------|------|
| **통신 방식** | Ethernet UDP 브로드캐스트 (자체 메시지 포맷) | 무선 기반 C-V2X / DSRC |
| **통신 범위** | CANoe 시뮬레이션 환경 내 지연/손실이 통제된 안정적 환경 | 물리적 노이즈가 존재하는 실도로망 |
| **제어 구조** | Ethernet → CAN 게이트웨이 변환 | 실제 OBU/RSU 인프라를 거친 변환 |
| **프로토콜** | 자체 정의 메시지 포맷 | 국제 표준 V2X 프로토콜 (DENM, CAM 등) |
| **확장성** | 하위 통신 모듈 교체(Plug-in) 구조 확보 | 표준(ASN.1) 라우팅 스택 및 OBU 하드웨어 이식 |

이 구현은 **아키텍처 개념 검증(PoC)** 수준이며, 실제 V2X 표준 프로토콜로의 확장이 자연스러운 다음 단계입니다.
이러한 물리적 차이점에도 불구하고, 본 프로젝트를 통해 검증된 **상위 애플리케이션 제어 로직과 CAN 라우팅 아키텍처**는 통신 매체에 종속되지 않습니다. 물리적인 OBU/RSU 하드웨어 파츠만 결합되면 즉시 작동할 수 있는 견고한 베이스라인 메커니즘을 제공한다는 점에서 본 PoC의 기술적 가치가 큽니다.

## 2. 기술 고도화 방향 — CANoe Car2x 라이선스 적용

Vector CANoe의 **Car2x 확장 라이선스**를 추가하면, 현재 Ethernet UDP 메시지를 실제 국제 표준 V2X 프로토콜로 대체할 수 있습니다. 차량 노드 구조와 내부 CAN 제어 흐름은 변경 없이 유지되며, **메시지 전송 계층만 표준 프로토콜로 교체**됩니다.

### 적용 가능한 표준 V2X 프로토콜

| 프로토콜 / 표준 | 계층 | 역할 | 현재 구현과 대응 |
|----------------|------|------|-----------------|
| **ITS-G5** (IEEE 802.11p) | 무선 MAC/PHY | 차량 간 직접 통신 (DSRC 기반 5.9GHz 대역) | 현재 Ethernet UDP → ITS-G5 무선 채널로 교체 |
| **C-V2X** (3GPP PC5 / 5G NR V2X) | 무선 MAC/PHY | 셀룰러 기반 차량 통신 (LTE/5G 활용) | 위성 백홀과 결합 시 전장 환경 재현 가능 |
| **CAM** (Cooperative Awareness Message) | 애플리케이션 | 주변 차량에 위치·속도·상태 주기 브로드캐스트 | 경찰차·선두차의 긴급 브로드캐스트와 직접 대응 |
| **DENM** (Decentralized Environmental Notification Message) | 애플리케이션 | 위험 이벤트 발생 시 주변에 경고 전파 | 시나리오 1·2의 긴급/위협 메시지와 직접 대응 |
| **ETSI ITS** (EN 302 665 등) | 전체 스택 | 유럽 C-ITS 표준 전체 프로토콜 스택 | ITS-G5 + CAM/DENM 포함한 통합 표준 |
| **SAE J2735 / J2945** | 애플리케이션 | 미국 DSRC 기반 BSM (Basic Safety Message) | 북미 표준 — CAM과 동일 역할 |

### 적용 시 구조 변화

```
[현재]
Police_Node → Ethernet UDP → Civ_Node
                (자체 포맷)

[Car2x 라이선스 적용 후]
Police_Node → ITS-G5 / C-V2X → Civ_Node
              (DENM 표준 메시지)
                    ↓
          ETSI ITS 전체 스택 준수
          (GeoNetworking → BTP → DENM)
```

차량 노드 내부 CAN 제어 로직은 변경 없이 유지되며, Ethernet 메시지 전송 부분만 표준 프로토콜로 교체됩니다.

## 3. 추가 연구 방향 — V2R (Vehicle-to-Robotics)

본 프로젝트의 군집 통신망은 차량에 국한되지 않고 **무인 플랫폼으로의 확장**이 가능합니다.

전장에서의 최신 트렌드인 **드론(Drone) 연동**과 **무인 지상 차량(UGV, Unmanned Ground Vehicle)** 운용이 대표적입니다.

- **드론 연동 (V2D)**: 선두 차량이 정찰 드론을 전방에 띄워 위협 요소(장애물, 매복)를 스캔하고, 그 데이터를 군집 통신망을 통해 후행 차량 전체에 실시간 공유.
- **무인 자율 로봇 (UGV)**: 전장이나 재난 현장에서 선두 차량을 무인 UGV로 전환하고, 후방 유인 통제 차량이 V2V 통신망을 통해 조향·제동을 원격 종속 제어(Master-Slave).

이 플랫폼의 1:N 동기 명령 구조는 차량뿐 아니라 드론·로봇까지 동일한 통신 아키텍처로 수용할 수 있어, V2R 연구의 기반으로 활용 가능합니다.

---

## 4. 연구 제언

> 본 프로젝트의 Ethernet-CAN 하이브리드 V2X 검증 플랫폼은 아키텍처 개념 검증(PoC) 단계를 충실히 완료했습니다.
>
> 다음 단계로 CANoe Car2x 라이선스 환경에서 ITS-G5, C-V2X, CAM, DENM 등 실제 표준 프로토콜을 적용하면 실도로 수준의 특수목적 차량 V2X 검증 시스템으로 발전할 수 있으며, 나아가 드론·UGV와 같은 이기종 무인 플랫폼까지 동일한 통신 아키텍처로 확장하는 V2R 연구의 출발점이 될 수 있습니다.
