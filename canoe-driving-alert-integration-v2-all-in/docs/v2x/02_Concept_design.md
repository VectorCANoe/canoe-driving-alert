# 콘셉트 설계서

| 구분 | 내용 |
|------|------|
| **프로젝트명** | 특수목적 차량 V2V 통신 검증 플랫폼 |
| **핵심 컨셉** | 차량들이 Ethernet으로 서로 직접 대화한다. 경찰차는 주변 차량에게 길을 요청하고, 군용차는 위협에 일사불란하게 반응하며, 물류차는 재난 순간 긴급차량으로 변신한다 |
| **타겟** | 공공 안전 인프라 / 국방·재난 대응 특수목적 차량 (PBV: Purpose Built Vehicle) |
| **주요 기능** | ① 경찰·응급차 → Ethernet 브로드캐스트 → 주변 차량 HMI 강제 경고 (긴급 통행권)<br>② 선두 군용차 → Ethernet 동기 명령 → 군집 전체 동시 회피·스텔스 (위협 대응)<br>③ 국가 서버 → Ethernet OTA → 물류차 긴급차량 모드 전환 (임무 변환) |
| **통신 구조** | CAN: 차량 내부 ECU 제어 / Ethernet: 차량 간 V2V 통신 및 OTA 배포 |
| **기술 목표** | 멀티 노드 V2V 브로드캐스트 실증. 1대 송신 → N대 동시 반응 아키텍처 검증. Ethernet 기반 펌웨어 전송 + CRC 검증 + Fail-Safe 동작 검증 |
| **비즈니스 근거** | 현대차 PBV 플랫폼, 국방부 자율주행 군용차, ETSI ITS 긴급차량 통신 표준(ITS-G5) 등 실제 산업 방향과 일치 |

## 시나리오 요약

```
[시나리오 1] 경찰차 긴급 통행권
  경찰차 → Ethernet 브로드캐스트 → 일반차 A / B / C
  동시에: 계기판 팝업 "긴급 차량 접근" + 앰비언트 주황 점멸

[시나리오 2] 군집 위협 대응
  선두 군용차 → Ethernet 동기 명령 → 후행차 1 / 2 / 3
  Level 1: 경계등 점멸
  Level 2: 비상 제동 동시 실행
  Level 3: 전 조명 소등 (스텔스)

[시나리오 3] 물류차 임무 전환 OTA
  국가 서버 → Ethernet OTA → 물류차
  펌웨어 수신 → CRC 검증 → 재시작 → 긴급차량 모드
```
