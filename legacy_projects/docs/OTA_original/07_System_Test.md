## Part 1. Base Setup 시나리오 (공통 — Scene.B)

> 운전자의 실제 주행 행동과 차량의 반응을 중심으로 서술한다.
> 이 시나리오가 전부 Pass되면 기본 차량 경고 시스템이 완성된 것이다.

| Scene. ID | 대응 Req | 설명 | Pass/Fail | 담당자 | 일자 |
|-----------|---------|------|----------|--------|------|
| **Scene.B1** | Req_B01~B03 | Power OFF 상태로 테스팅을 시작한다. 계기판이 꺼져 있고 경고등, 앰비언트 라이트, 경고음이 모두 없는 것을 확인한다. | | | |
| **Scene.B2** | Req_B01~B03 | Power ON으로 전환한다. 계기판이 켜지며 속도 0, 경고 없음, 기어 P 상태가 표시되는 것을 확인한다. 앰비언트 라이트는 꺼진 상태이다. | | | |
| **Scene.B3** | Req_B01, B04, B09 | 기어를 D로 변경하고 가속 페달을 밟아 속도를 60km/h까지 올린다. 제한속도(80km/h) 이내이므로 계기판에 경고등이 없고, 앰비언트 라이트는 꺼진 상태이며, 경고음도 울리지 않는 것을 확인한다. | | | |
| **Scene.B4** | Req_B04, B09, B10~B13 | 가속 페달을 밟아 속도를 90km/h까지 올린다. 일반도로 제한속도(80km/h)를 초과한 순간 계기판에 황색 경고등이 순간 켜지고, "띠딩!" 단발음이 한 번 울리며, 화면에 "주의" 메시지가 표시되는 것을 확인한다. 경고 발령 후 2초가 지나면 운전자 조작 없이 경고등과 메시지가 자동으로 꺼지는 것을 확인한다. | | | |
| **Scene.B5** | Req_B05, B09, B10~B13 | 속도 60km/h 정상 주행 중 가속 페달을 급하게 밟아 급가속 상황을 만든다. 계기판에 황색 경고등이 순간 켜지고 "띠딩!" 단발음이 한 번 울린다. 2초 후 자동 소거되는 것을 확인한다. | | | |
| **Scene.B6** | Req_B06, B09, B10~B13 | 고속 주행 중 브레이크를 급하게 밟아 급제동 상황을 만든다. 계기판에 황색 경고등이 순간 켜지고 "띠딩!" 단발음이 한 번 울린다. 2초 후 자동 소거되는 것을 확인한다. | | | |
| **Scene.B7** | Req_B07, B09, B10~B13 | 정상 주행 중 차선을 이탈하는 상황이 발생한다. 계기판에 황색 경고등이 순간 켜지고 "띠딩!" 단발음이 한 번 울린다. 2초 후 자동 소거되는 것을 확인한다. | | | |
| **Scene.B8** | Req_B08, B09, B10~B13 | 정상 주행 중 갑작스러운 차선 변경이 발생한다. 계기판에 황색 경고등이 순간 켜지고 "띠딩!" 단발음이 한 번 울린다. 2초 후 자동 소거되는 것을 확인한다. | | | |
| **Scene.B9** | Req_B04~B09, B10~B13 | 속도를 90km/h까지 올린 상태에서 차선이탈이 동시에 발생한다. 계기판 경고등이 황색에서 적색으로 바뀌고, 경고음이 500ms 간격 연속음으로 바뀌며, 앰비언트 라이트가 주황색 파동 패턴으로 켜지는 것을 확인한다. 화면에 경고 원인(과속+차선이탈)이 함께 표시되는 것을 확인한다. 과속 또는 차선이탈 중 하나가 해소되면 경고가 자동으로 Level 1로 낮아지거나 소거되는 것을 확인한다. | | | |
| **Scene.B10** | Req_B09, B10~B13 | Panel에서 충돌 이벤트(gCrashEvent) 버튼을 누른다. 계기판 적색 경고등이 빠르게 점멸하고, 긴급 경고음이 지속적으로 울리며, 앰비언트 라이트가 빨간색으로 빠르게 점멸하는 것을 확인한다. **3단계 경고는 시간이 지나도 자동 소거되지 않는 것을 확인한다.** | | | |
| **Scene.B11** | Req_B15 | **3단계 경고(Scene.B10) 상태에서** 운전자가 전방을 바라본다. 모든 경고등이 꺼지고, 경고음이 멈추며, 앰비언트 라이트가 꺼지는 것을 확인한다. 화면이 정상 주행 상태로 복귀하는 것을 확인한다. | | | |
| **Scene.B12** | Req_B16 | **3단계 경고(Scene.B10) 상태에서** 운전자가 핸들을 조작한다. 모든 경고등이 꺼지고, 경고음이 멈추며, 앰비언트 라이트가 꺼지는 것을 확인한다. | | | |

> **Base Setup 통과 기준**: Scene.B1~B12 전체 Pass.

## Part 2. Test Suite 1 — 준영 시나리오 (구간 인식 + 앰비언트 연동 — Scene.Z)

> **전제조건**: Scene.B1~B12 전체 Pass.
> 운전자가 네비게이션을 통해 특정 구간에 진입하는 상황을 시뮬레이션하며, 앰비언트 라이트가 구간에 맞게 반응하는지 확인한다.

| Scene. ID | 대응 Req | 설명 | Pass/Fail | 담당자 | 일자 |
|-----------|---------|------|----------|--------|------|
| **Scene.Z1** | Req_Z01, Z02 | 스쿨존 구간에 진입한다. 속도를 35km/h로 주행한다(스쿨존 제한 30km/h 초과). 계기판에 황색 경고등이 켜지는 동시에 앰비언트 라이트가 빨간색으로 빠르게 점멸하며 어린이 보호구역 과속 경고를 표시하는 것을 확인한다. 속도를 25km/h로 줄이면 앰비언트 경고가 해제되는 것을 확인한다. | | | |
| **Scene.Z2** | Req_Z01, Z03 | 고속도로 구간에 진입하여 110km/h로 주행한다. 핸들 조작 없이 직진 주행을 10초간 유지한다. 10초가 지나는 순간 앰비언트 라이트가 주황색 파동 패턴으로 바뀌고 시트에 진동이 발생하여 전방주시 태만 경고를 표시하는 것을 확인한다. 핸들을 조작하면 경고가 즉시 해제되는 것을 확인한다. | | | |
| **Scene.Z3** | Req_Z01, Z04 | 고속도로 IC 출구 근접 구간에 진입한다. Panel에서 진출 방향(좌/우)을 설정한다. 앰비언트 라이트가 실도로 IC 진출입 유도선 색상(초록/분홍/빨강)으로 즉시 전환되고, 진출 방향으로 흐르는 것을 확인한다. 구간을 벗어나면 앰비언트가 즉시 꺼지는 것을 확인한다. | | | |
| **Scene.Z4** | Req_Z01~Z04 | 일반도로→스쿨존→고속도로→IC출구→일반도로 순서로 구간을 연속 전환한다. 각 구간 진입 시 앰비언트 패턴이 즉시 해당 구간 패턴으로 전환되고, 이전 구간 패턴이 완전히 사라지는 것을 확인한다. | | | |

## Part 3. Test Suite 2 — 성현 시나리오 (Drive Coach + Smart Claim + Seasonal Theme — Scene.O)

> **전제조건**: Scene.B1~B12 전체 Pass.
> 동일한 UDS OTA 파이프라인으로 3종 패키지(Drive Coach / Smart Claim / Seasonal Theme)를 순서대로 검증한다.

| Scene. ID | 대응 Req | 설명 | Pass/Fail | 담당자 | 일자 |
|-----------|---------|------|----------|--------|------|
| **Scene.O1** | Req_O01 | 차량을 P 기어 상태로 정차한다. IVI 화면에서 구독 메뉴를 열고 Drive Coach Package [설치] 버튼을 선택한다. 운전자 동의 화면이 표시되고, 동의하면 OTA 다운로드가 시작되는 것을 확인한다. P 기어 이탈 시 OTA 중단 여부도 확인한다. | | | |
| **Scene.O2** | Req_O01 | 시스템이 보안 인증(0x27)을 진행하고 패키지를 전송한다. 진행률 0%→100%, CRC 검증 통과, 차량 재시작. 재시작 후 Drive Coach 기능이 활성화된 것을 확인한다: 급차선변경 시 LDW 경고 강화 점등, 급가속 시 토크 제한 적용(가속 반응 제한), 최고속도 100 km/h 리미터 작동, 후진 시 10 km/h 초과 불가. | | | |
| **Scene.O3** | Req_O02 | Drive Coach 설치 완료 후 Smart Claim이 활성화된 상태에서 gCrashEvent = 1을 다시 발생시킨다. Python CANoe COM API가 충돌 데이터(타임스탬프, 속도, 경고 이력)를 Flask 서버로 전송하는 것을 확인한다. 화면에 "사고 데이터 전송 완료" 메시지가 표시되는 것을 확인한다. | | | |
| **Scene.O4** | Req_O03 | OTA 서버에서 시즌 테마 패키지가 가용함을 알린다. 화면에 업데이트 알림이 표시되고, 운전자 동의 후 동일한 UDS 세션 흐름으로 패키지가 설치되는 것을 확인한다. 설치 완료 즉시 앰비언트 색상 프리셋과 IVI 배경이 변경되는 것을 확인한다. | | | |
| **Scene.O5** | Req_O04 | OTA 진행 중 데이터 오류를 발생시킨다. 차량이 자동으로 이전 버전으로 복구하고 화면에 "업데이트 실패, 이전 버전으로 복구됨" 메시지가 표시되는 것을 확인한다. | | | |
| **Scene.O6** | Req_O05 | OTA 진행 중 CGW Bus Off를 발생시킨다. 차량이 업데이트를 즉시 중단하고 화면에 "통신 오류, 세션 중단" 메시지가 표시되는 것을 확인한다. 기본 경고 시스템이 정상 동작하는 것을 확인한다. | | | |

## 시나리오 구조 요약

```
[Base Setup — Scene.B1~B12]
  B1. Power OFF 시작
  B2. Power ON → 초기 상태 확인
  B3. 정상 주행 진입 (60km/h, 경고 없음)
  B4. 과속 → 1단계 경고 (황색 순간점등 + "띠딩!" 단발음) → 2초 자동 소거
  B5. 급가속 → 1단계 경고 → 2초 자동 소거
  B6. 급제동 → 1단계 경고 → 2초 자동 소거
  B7. 차선이탈 → 1단계 경고 → 2초 자동 소거
  B8. 급차선변경 → 1단계 경고 → 2초 자동 소거
  B9. 과속 + 차선이탈 동시 → 2단계 경고 (적색 + 연속음 + 앰비언트 주황) → 조건 해제 시 자동 소거
  B10. gCrashEvent = 1 → 3단계 경고 (적색 점멸 + 긴급음 + RED 앰비언트) → 자동 소거 없음
  B11. [3단계 상태에서] 운전자 전방 응시 → 경고 해제
  B12. [3단계 상태에서] 운전자 핸들 조작 → 경고 해제
          │
          ├─ [Test Suite 1 — Scene.Z1~Z4] gRoadZone 패널 버튼만 추가
          │    Z1. 스쿨존 진입 → 과속 시 앰비언트 빨간 점멸
          │    Z2. 고속도로 진입 → 핸들 미조작 10초 → 앰비언트 주황 파동 + 시트 진동
          │    Z3. IC출구 진입 → 앰비언트 방향 안내 흐름
          │    Z4. 구간 연속 전환 → 패턴 즉시 전환 확인
          │
          └─ [Test Suite 2 — Scene.O1~O6] 3종 OTA 패키지 + OTA 동의 버튼만 추가
               O1. 3단계 경고 → Drive Coach 팝업 → 운전자 동의
               O2. UDS 세션 → 펌웨어 전송 → 재시작 → Drive Coach 적용 확인
               O3. Smart Claim 활성화 → 충돌 이벤트 → Flask 서버 데이터 전송 확인
               O4. Seasonal Theme OTA → 설치 → 즉시 적용 확인
               O5. 데이터 오류 → 자동 이전 버전 복구
               O6. Bus Off → 세션 중단 → 기본 시스템 정상 유지
```
