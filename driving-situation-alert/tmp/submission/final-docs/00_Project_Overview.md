## 1. 문서 목적

본 문서는 제출본 전체(00~07)의 공통 배경을 한 페이지로 요약한다.
프로젝트 목적, 핵심 시나리오, 구현 범위, 문서 구성을 간결하게 정리한다.

## 2. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 개발 목적 | 주행 구간 정보와 긴급차량 접근 정보를 통합해 운전자 경고를 실시간으로 제공한다. |
| 시스템 범위 | CANoe SIL 환경에서 Domain CAN + Ethernet Backbone 기반 경고 시스템을 구현한다. |
| 핵심 입력 | 차량 상태, 구간/경로 정보, 긴급차량 접근 정보, 객체/주행 위험 관련 상태 |
| 핵심 처리 | 경고 우선순위 판정, 경고 단일 결정, TTC 기반 충돌 위험 판단, 자동 감속 보조 요청, 경고 채널 동기화, 장애 시 fail-safe 처리 |
| 핵심 출력 | 클러스터, 앰비언트, 오디오 경고 출력, 자동 감속 보조 요청 |
| 검증 범위 | Unit/Integration/System 테스트를 통해 요구사항 충족 여부를 검증한다. |

## 3. 핵심 시나리오

| 시나리오 | 설명 | 관련 문서 |
|---|---|---|
| 구간 기반 경고 | 일반/스쿨존/고속/유도 구간 상태에 따라 경고 정책을 적용한다. | `01`, `03`, `07` |
| 긴급차량 접근 경고 | 경찰/구급 긴급 알림 수신 시 우선 경고를 표시하고, 동시 접근 시 구급 경고를 우선한다. | `01`, `03`, `07` |
| 교차로/합류 TTC 감속 보조 | 교차로 및 합류 구간에서 긴급차량과의 TTC 충돌 위험이 높아지면 자동 감속 보조를 요청한다. | `01`, `03`, `07` |
| 경고 충돌 및 복귀 | 복수 경고 충돌 시 우선순위 규칙으로 단일 경고를 선택하고, 종료 시 안전하게 복귀한다. | `01`, `04`, `07` |

## 4. 핵심 용어

| 용어 | 설명 |
|---|---|
| Emergency 경고 | 긴급차량 접근(예: 구급차, 경찰차) 상황에서 운전자에게 즉시 제공되는 우선 경고를 의미한다. |
| Zone 경고 | 스쿨존, 고속도로, 유도차선 등 주행 구간 특성에 따라 제공되는 일반 경고를 의미한다. |
| ETA | 긴급차량 후보별 도착예정시간 값(초)이다. 값이 작을수록 먼저 도달하는 후보로 판단한다. |
| SourceID | 동일 긴급차량 유형 후보를 구분하는 송신원 식별값이다. ETA가 같으면 값이 작은 후보를 우선한다. |
| 단일 경고 결정 | 복수 경고 후보가 동시에 존재할 때 우선순위, 긴급차량 유형, ETA, SourceID 순서로 최종 경고를 결정하는 규칙이다. |

## 5. 설계 규칙

| 규칙 | 내용 |
|---|---|
| 우선순위 | Emergency 경고가 Zone 경고보다 우선한다. |
| 긴급차량 우선 | Ambulance가 Police보다 우선하며, 동시 접근 시 동일 규칙을 적용한다. |
| 단일 경고 결정 기준 | 경고 후보가 복수인 경우 정의된 우선순위 규칙으로 단일 경고를 결정한다. |
| 감속 보조 기준 | 교차로 및 합류 구간에서 긴급차량과의 TTC 충돌 위험이 기준 이상이면 자동 감속 보조를 요청한다. |
| 복귀 기준 | 긴급 입력 1000ms 무갱신 시 경고를 해제하고 정상 상태로 복귀한다. |
| 계층 분리 | 제품 기능 계층과 검증 계층을 분리해 운영한다. |

## 5-1. 단일 경고 결정 매트릭스

| 단계 | 비교 조건 | 결정 규칙 |
|---|---|---|
| 1 | Emergency 경고 vs Zone 경고 | Emergency 경고를 우선 선택한다. |
| 2 | Ambulance vs Police | Ambulance를 우선 선택한다. |
| 3 | 동일 긴급차량 유형 후보 간 비교 | ETA가 더 작은 후보를 우선 선택한다. |
| 4 | 동일 긴급차량 유형, 동일 ETA 후보 간 비교 | SourceID가 더 작은 후보를 우선 선택한다. |
| 5 | 동일 긴급차량 유형, 동일 SourceID 후보 재수신 | 동일 송신원의 갱신으로 보고 방향/ETA를 최신 값으로 갱신한다. |
