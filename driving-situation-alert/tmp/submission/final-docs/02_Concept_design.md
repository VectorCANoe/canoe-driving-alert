## 1. 문서 목적

본 문서는 주행 상황 실시간 경고 시스템의 전체 구조와 핵심 동작 개념을 설명한다.
제출본에서는 차량 관점의 구조, 주요 규칙, 대표 시나리오만 간단히 정리한다.

## 2. 기준 아키텍처

- 채택안: `Option 1`
- 구조: `ETH_BACKBONE + CGW + Domain CAN + 중앙 경고 판단 + HMI 출력`
- 범위: `CANoe SIL`, `CAN + Ethernet(UDP)`
- 문서 표기: 상위 설명은 표면 ECU 기준으로 읽는다.
- 검증 분리: Validation Harness는 제품 기능 계층과 분리한다.

주요 규칙:
- `Emergency > Zone`
- `Ambulance > Police`
- 동률 시 `ETA 오름차순 -> SourceID 오름차순`
- `1000ms` 무갱신 시 경고를 해제하고 정상 상태로 복귀한다.

## 3. 핵심 동작

| 구분 | 내용 |
|---|---|
| 구간 기반 경고 | 스쿨존, 고속도로, 유도차선 등 주행 구간 정보를 받아 운전자 경고를 표시한다. |
| 긴급차량 기반 경고 | 경찰차, 구급차 접근 정보를 받아 우선 경고를 표시한다. |
| 경고 중재 | 구간 경고와 긴급 경고가 동시에 발생하면 우선순위 규칙에 따라 한 개의 경고를 선택한다. |
| 출력 동기화 | 선택된 경고는 앰비언트, 클러스터, 오디오 채널에 맞춰 함께 반영한다. |
| 복귀 동작 | 경고 조건이 해제되면 이전 주행 문맥으로 안정적으로 복귀한다. |

## 4. 구성 원칙

- 입력은 차량 상태, 내비 문맥, 긴급 알림으로 나눈다.
- 판단은 중앙 경고 기능에서 수행한다.
- 출력은 앰비언트, 클러스터, 오디오 경고로 반영한다.
- 상세 알고리즘과 구현 예외는 `04`에서 관리한다.
- 네트워크와 통신 정의는 `0302`, `0303`, `0304`에서 관리한다.
