## 1. 문서 목적

본 문서는 프로젝트의 전체 구조, 도메인 경계, 핵심 경고 흐름을 그림 중심으로 제시한다.
이 문서에서는 구현 세부보다 시스템 구성과 경고 전달 구조를 우선 설명한다.

## 3. 그림 구성 및 해석 포인트

| 그림 번호 | 그림명 | 확인 항목 | 의미 |
|---|---|---|---|
| 02-01 | 전체 구조도 | 입력, 판단, 출력, 검증 계층이 분리되어 있는지 | 차량 상태, 구간 정보, 긴급차량 정보를 받아 중앙 경고 기능이 최종 경고를 결정하고, 이를 운전자 출력 채널로 전달하는 전체 구조를 보여준다. |
| 02-02 | 네트워크 흐름도 | Domain CAN과 Ethernet Backbone의 경계가 명확한지 | 차량 입력은 도메인 CAN에서 수집하고, 도메인 간 전달은 Ethernet 백본을 통해 수행하며, 최종 경고는 출력 도메인으로 반영되는 통신 구조를 보여준다. |

## 4. 주요 확인 포인트

| 항목 | 설명 |
|---|---|
| 시스템 구조 | 입력 계층, 판단 계층, 출력 계층, 검증 계층이 역할별로 분리되어 있는지 확인한다. |
| 경고 흐름 | 구간 경고와 긴급차량 경고가 동일 시스템 안에서 처리되는지 확인한다. |
| 도메인 경계 | Domain CAN과 Ethernet Backbone의 연결 구조가 명확한지 확인한다. |
| 출력 일관성 | 최종 경고가 클러스터와 앰비언트 중심으로 일관되게 전달되는지 확인한다. |
| 검증 구조 | 검증 계층이 제품 기능 계층과 분리되어 있는지 확인한다. |

## 5. 핵심 설계 규칙

- 범위: `CANoe SIL`, `CAN + Ethernet(UDP)`
- 구조: `Ethernet Backbone + Domain CAN + 중앙 경고 판단 + HMI 출력`
- 우선순위: `Emergency > Zone`
- 긴급차량 우선순위: `Ambulance > Police`
- 동률 처리: `ETA 오름차순 -> SourceID 오름차순`
- 복귀 기준: `1000ms` 무갱신 시 경고를 해제하고 정상 상태로 복귀한다.
