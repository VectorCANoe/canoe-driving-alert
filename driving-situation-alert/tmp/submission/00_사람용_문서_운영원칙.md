# 사람용 문서 운영 원칙 (문서팀)

## 1) 기본 원칙
- 사람이 읽는 설명 문장은 한글로 작성한다.
- 코드/신호/변수/ID/파일명 같은 식별자는 원문(영문) 그대로 둔다.
- 본문은 짧고 명확하게 유지하고, 상세 근거는 부록/링크로 분리한다.

## 2) 한글 적용 대상
- 제출본 본문: `tmp/submission/final-docs/*.md`
- 제출 안내/운영 문서: `tmp/submission/*.md`
- 멘토/팀 커뮤니케이션 문서(문장 설명 파트)

## 3) 영문 유지 대상
- Req/Func/Flow/Comm/Var 식별자
- CAN ID, 메시지명, SysVar명, ECU 노드명
- 스크립트/게이트 명령어, 파일 경로

## 4) 작성 스타일
- 용어는 프로젝트 표준 용어만 사용한다.
- 한 문단에 하나의 의사결정만 적는다.
- 표는 “결정/근거/영향” 3열 우선으로 간단히 유지한다.
