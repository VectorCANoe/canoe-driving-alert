## 1) 프로젝트 한 줄 정의

주행 중 운전자에게 필요한 경고를 "상황 맥락 + 긴급도" 기준으로 실시간 선택해, 앰비언트와 클러스터에 일관되게 전달하는 통합 경고 시스템을 구현한다.

## 2) 왜 필요한가 (문제 정의)

- 운전자는 동시에 여러 경고가 들어올 때 무엇을 먼저 봐야 하는지 판단하기 어렵다.
- 긴급차량 접근, 구간 규제(스쿨존/고속도로), 일반 주행 경고가 섞이면 경고 과밀과 혼선이 발생한다.
- 따라서 "경고를 많이 만드는 것"보다 "우선순위를 정해 하나의 최종 경고로 정리하는 것"이 핵심이다.

## 3) 우리가 제안한 해결 컨셉

### 컨셉 A: 입력을 분리해서 받는다

- 차량 상태 입력(속도/조향/페달)
- 내비게이션 맥락 입력(구간/방향/거리/제한속도)
- 긴급차량 이벤트 입력(경찰/구급, ETA, 방향)

### 컨셉 B: 중앙에서 하나의 경고로 결정한다

- 중앙 경고코어(`ADAS_WARN_CTRL`, `NAV_CTX_MGR`, `EMS_ALERT`, `WARN_ARB_MGR`)에서 최종 경고 1건을 선택한다.
- 판단 기준은 "경보 우선순위 판정" 규칙으로 고정한다.

### 컨셉 C: 출력 채널을 동기화한다

- 최종 경고를 Body/IVI 경로로 분기해 앰비언트(`AMBIENT_CTRL`)와 클러스터(`CLU_HMI_CTRL`)에 동일 맥락으로 표시한다.

## 4) 핵심 시나리오

1. 스쿨존 과속 상황  
속도/제한속도 조건이 성립하면 경고가 즉시 활성화되고, 해제 조건 성립 시 정상 상태로 복귀한다.

2. 긴급차량 접근 상황  
긴급 이벤트 수신 시 일반 경고보다 우선 경고를 적용하고, 종료/타임아웃 규칙으로 안전하게 해제한다.

3. 경고 충돌 상황  
구간 경고와 긴급 경고가 동시에 들어오면 우선순위 규칙으로 단일 최종 경고를 산출한다.

## 5) 구현/검증 접근 방식

- 환경: CANoe SIL (CAN + Ethernet)
- 추적 체인: `Req -> Func -> Flow -> Comm -> Var -> Code -> UT/IT/ST`
- 현재 단계:
  - Active 범위는 구현/문서/테스트를 연결해 운영
  - Pre-Activation 범위는 설계 선반영 후 상태를 명시적으로 관리

## 6) 기대 결과 (중간 제출 관점)

- 기능 측면: 핵심 경고 시나리오에서 "단일 최종 경고 결정" 동작을 설명 가능
- 구조 측면: 01~07 문서 체인으로 설계 의도와 구현 경로를 역추적 가능
- 품질 측면: SIL 환경에서 테스트 재현성과 검증 가능성을 확보
- 확장 측면: Ethernet 정식 전환 시 동일 구조를 유지한 채 네트워크 계층만 단계적 교체 가능

## 7) 범위 경계

### Scope In

- 구간 인식 경고, 긴급차량 경고, 경보 우선순위 판정, 출력 동기화, SIL 검증 체계

### Scope Out

- OTA/UDS/DoIP 통신 고도화, 실차/HIL 연동, 범위 외 레거시 위험점수 기능
