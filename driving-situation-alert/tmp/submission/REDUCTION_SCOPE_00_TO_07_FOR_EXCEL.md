# 00~07 제출용 축소 기준서 (복제본 편집 + 엑셀화 기준)

목표:
- `driving-situation-alert/00~07` 원문 SoT를 직접 제출하지 않고, 제출용 복제본에서 핵심만 남긴다.
- 비교 평가에 필요한 표 중심 구조를 유지한다.
- `Req -> Func -> Flow -> Comm -> Var -> Code -> UT/IT/ST` 추적 체인의 골격은 유지한다.

전제:
- 본 기준은 **제출용 복제본 전용**이다.
- 원본 SoT(00~07)는 그대로 보존한다.
- 실측 미실시 상태이므로 `05/06/07`의 Pre-Activation은 `Planned/Ready` 유지.

---

## 1) 공통 편집 규칙 (모든 문서 공통)

남긴다:
1. 문서 헤더(`Document ID`, `Version`, `Date`, `Status`)  
2. 공식 표(요구/기능/통신/변수/테스트)  
3. 추적 체인에 직접 쓰이는 매핑 표(Req-Func, Flow-Comm 등)  
4. 제출 판단에 필요한 정책 문장(짧게)

뺀다:
1. 장문 설명/배경/운영 메모
2. 하단 보강표 중 중복 정보
3. 체크포인트/운영 가이드/내부 작업 절차
4. 상세 개정 이력(최종 1~2개만 남기거나 전부 제거)

형식 규칙:
1. 표 1개에 동일 의미가 중복되면 하나만 유지
2. Legacy는 상세표 대신 “매핑 원칙 1줄 + 대표 매핑 3~5건”으로 축소
3. Pre-Activation은 표 비고에 라벨만 유지

## 4) 복제본 작업 순서 (권장)

1. `submission-copy` 폴더에 00~07 복제
2. 위 기준대로 섹션 삭제/축약
3. 남은 표의 열 순서를 샘플 엑셀 시트와 맞춤
4. Pre-Activation 상태 라벨 일관화(`Planned`/`Ready`)
5. 마지막에 스크립트로 엑셀 변환

주의:
- 원본 SoT 파일은 수정하지 않는다.
- 제출용 복제본에서만 축소 작업을 수행한다.
