## 0. 현재 상태

- G1 Mapping Freeze: Closed
- G2 Implementation Freeze: Closed
- G3 Cutover Approval: Closed
- G4 Post-Cutover Audit: Hold

본 문서는 전면 재할당 실행 지시서가 아니라, **현재 남은 운영/감사 항목**만 전달한다.

## 1. 고정 운영 규칙 (유지)

1. 도메인/게이트웨이 구조는 유지한다.
2. CAN ID 정책은 `00f_CAN_ID_Allocation_Standard.md`를 SoT로 사용한다.
3. 임의 재번호 부여(개별 패치식 ID 변경) 금지.
4. 추가 변경이 필요하면 CR 기반으로 Annex A 매핑표부터 갱신한다.

## 2. G4 보류 중 필수 관리 항목

1. 동결 유지
- G4 재개 전까지 `3/3/5` 매핑을 변경하지 않는다.
- DBC/CAPL/문서 중 어느 한쪽만 단독 변경하지 않는다.

2. 형상 일치 점검
- `Old ID` 잔존 참조 0건 유지
- 활성 DBC 간 중복 ID 0건 유지
- 문서 체인(`0302/0303/0304/04/05/06/07`)과 코드 체인 ID 불일치 0건 유지

3. 기준점 고정
- 롤백/기준점 기록은 기존 승인 커밋을 유지한다.
- 신규 기준점 생성 시 문서/코드 동시 갱신 후 기록한다.

## 3. G4 재개 조건 (전체 개발 종료 후)

1. 전체 기능 개발 완료(문서팀/개발팀 합의)
2. 최종 회귀 완료(핵심 시나리오 PASS)
3. Annex A 최종본 확정(매핑/근거/승인자/승인일)
4. G4 감사 체크리스트 실행 및 결과 기록

## 4. 개발팀 제출물 (G4 재개 시)

1. 최종 ID 충돌 검사 보고서
2. Old/New 잔존 참조 스캔 보고서
3. 핵심 시나리오 회귀 결과(UT/IT/ST 요약)
4. 최종 기준 커밋 해시 및 태그(있으면 포함)

## 5. 전달 메모

- 현재는 `운영 안정 구간`이며, 전면 재할당 재작업을 지시하지 않는다.
- 변경 필요 시 반드시 “매핑표 선행 -> 구현 반영 -> 증적 제출” 순서를 따른다.
