## 1. 문서 목적

본 문서는 시스템의 전체 구성과 네트워크 경계를 그림 중심으로 설명한다.
제출본에서는 구현 상세 대신 시스템 구조와 동작 원칙을 간결하게 제시한다.

## 2. 시스템 개요

| 항목 | 내용 |
|---|---|
| 시스템 목적 | 주행 구간 정보와 긴급차량 접근 정보를 통합하여 운전자 경고를 제공한다. |
| 입력 정보 | 차량 상태, 구간/경로 정보, 긴급차량 접근 정보, 객체/주행 위험 정보 |
| 판단 기능 | 경고 우선순위 규칙과 TTC 충돌 위험 판단을 반영해 최종 경고와 자동 감속 보조 요청 여부를 결정한다. |
| 출력 채널 | 클러스터, 앰비언트, 전면 표시, 오디오, 자동 감속 보조 요청, 외부 송신 경로 |
| 통신 구조 | 차량 CAN + Ethernet |
| 안전 동작 | 경고 전달 이상 시 최소 경고를 유지하고 fail-safe로 전환한다. |
| 검증 범위 | CANoe SIL, CAN + Ethernet(UDP) |

## 4. 그림 설명

| 02-01 | 전체 구조도 | 입력, 판단, 출력, 안전 동작 계층이 분리된 전체 시스템 구성을 보여준다. |

## 5. 설계 규칙

| 규칙 | 내용 |
|---|---|
| 우선순위 규칙 | 긴급 경고(Emergency)가 구간 경고(Zone)보다 우선한다. |
| 긴급차량 우선순위 | 구급차(Ambulance)가 경찰차(Police)보다 우선한다. |
| 동률 처리 | 우선순위가 같은 경고가 동시에 존재하면 ETA가 짧은 경고를 먼저 선택하고, ETA도 같으면 SourceID 오름차순으로 결정한다. |
| 감속 보조 기준 | 교차로 및 합류 구간에서 긴급차량과의 TTC 충돌 위험이 기준 이상이면 자동 감속 보조를 요청한다. |
| 송신 기준 | 긴급 이벤트가 발생하면 긴급차량 유형에 맞는 외부 송신 경로로 알림을 전달한다. |
| fail-safe 기준 | 경고 전달 이상 시 자동 감속 보조를 차단하고 최소 경고를 유지한 상태로 fail-safe 전환한다. |
| 복귀 기준 | 1000ms 동안 갱신이 없으면 경고를 해제하고 정상 상태로 복귀한다. |
