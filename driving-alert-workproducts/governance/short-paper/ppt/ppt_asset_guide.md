# PPT 이미지/도식 가이드

이 문서는 발표용 PDF 본문과 분리해, 실제 PPT 편집 시 어떤 이미지나 도식을 넣으면 좋은지 정리한다.

## 원칙

- 발표 PDF 본문에는 이미지 지시문을 직접 노출하지 않는다.
- 실제 PPT 편집 단계에서는 슬라이드당 1~2개의 시각자료만 사용한다.
- 그림은 `설명용`, `증거용`, `구조용` 중 하나의 역할만 하게 한다.

## 권장 이미지 매핑

### 대표 시나리오 end-to-end

- 추천 그림:
  - `입력 -> V2X 정규화 -> ADAS 중재 -> 출력 -> 시험` 흐름도 1장
- 목적:
  - 청중이 한 장에서 전체 동작을 이해하게 한다.

### 실행 증거와 관측 화면

- 추천 그림:
  - CANoe Test Unit 실행 화면 1장
  - 패널 또는 Diagnostic Console 캡처 1장
- 목적:
  - 실제 실행과 관측이 가능한 구조임을 보여준다.

### 산출물 패키지와 운영 워크스페이스

- 추천 그림:
  - `00~07 문서 + Test Matrix + CANoe 실행 + appendix bundle + Excel`을 연결한 패키지 구성도 1장
- 목적:
  - 코드뿐 아니라 운영 가능한 기준선을 구축했다는 점을 보여준다.

### 산업적 의미

- 추천 그림:
  - 기능 추가 중심 접근과 정책 통합 중심 접근을 비교하는 개념도
  - 또는 SDV/V2X 확장 로드맵 요약도
- 목적:
  - 기술 구현을 넘어 왜 이 구조가 의미가 있는지 보여준다.
