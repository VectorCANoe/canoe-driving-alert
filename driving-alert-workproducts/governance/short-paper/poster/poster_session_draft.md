# 포스터 세션 초안

## 제목
구간 정보와 긴급차량 접근을 통합한 실시간 주행 경고 컨셉  
부제: CANoe SIL 기반 구현, 시험 구조, 검증 운영 자동화

## 1. 문제 정의
핵심 컨셉은 스쿨존·고속도로·유도선 구간 경고, 긴급차량 접근 경고, 교차로·합류구간 충돌위험 경고를 하나의 정책으로 통합하는 실시간 주행 경고 기능이다. 개별 경고 기능을 분리해 운용할 경우 우선순위 충돌, 표시 중복, timeout 이후 복귀 불안정이 반복될 수 있어 이를 하나의 계층형 경고 중재 구조로 통합할 필요가 있었다.

## 2. 프로젝트 목표
현대 차량 시스템을 참고한 ECU 구성과 핵심 서비스 경로를 바탕으로 주행 경고 컨셉을 다시 구성하였다. 이를 CANoe SIL 환경에서 구현하고 반복 시험할 수 있는 구조로 구체화하였다.

[이미지 삽입 위치]
- 권장 이미지: 프로젝트 범위 인포그래픽
- 들어갈 내용: Hyundai-style ECU baseline, CAN/Ethernet, CANoe SIL, CAPL, UT/IT/ST
- 캡션 예시: `프로젝트 구현 및 시험 범위`

## 3. 시스템 구조
시스템은 입력 수집, 이벤트 정규화, 위험도 산정 및 경고 중재, HMI·감속 보조 출력, 검증 관측의 구조로 구성하였다. `CGW`, `IVI`, `V2X`가 입력을 수집하고 `ADAS`가 최종 경고와 감속 보조 요청을 결정하며, `BCM`, `CLU`, `HUD`, `AMP`가 결과를 반영한다. CAN은 차량 상태와 구간 맥락 수집에, Ethernet은 긴급 이벤트와 경고 전달에 사용하였다.

[이미지 삽입 위치]
- 권장 이미지: ECU 블록도 + CAN/Ethernet 흐름이 함께 보이는 구조도 1장
- 캡션 예시: `입력 수집, 경고 중재, 출력 반영, 검증 관측으로 구성된 시스템 구조`

## 4. 핵심 기능
핵심 기능은 V2X 기반 긴급 이벤트 정규화와 ADAS 기반 위험도 산정 및 경고 중재에 있다. V2X는 긴급차량 유형, ETA, SourceID를 기준으로 활성 이벤트를 선별하고 timeout clear로 이벤트 유효 상태를 관리한다. ADAS는 긴급 접근 정보와 TTC 기반 객체 위험을 결합해 최종 경고와 감속 보조 요청을 결정하며, on/off 안정화와 fail-safe를 통해 경고 전환과 복귀의 일관성을 유지하도록 설계하였다.

[이미지 삽입 위치]
- 권장 이미지: ADAS 위험도 산정 및 경고 중재 블록도
- 대안 이미지: V2X 이벤트 선택 상태도
- 캡션 예시: `V2X 정규화와 ADAS 경고 중재를 중심으로 구성된 핵심 기능`

## 5. 구현 및 구축 성과
요구사항, HARA, 기능 정의를 기준으로 검증 계약, Test Oracle, Master Test Matrix, UT/IT/ST 시험 자산을 구성하였다. 이를 통해 ISO 26262 추적성과 V-cycle 기반 검증 구조를 유지하고, 구간 경고 반영, 긴급차량 우선순위 판단, timeout clear, 교차로·합류구간 위험 경고를 핵심 검증 시나리오로 구성하였다.

[이미지 삽입 위치]
- 권장 이미지: CANoe Test Unit 실행 화면
- 대안 이미지: 운영 콘솔 TUI 화면
- 캡션 예시: `대표 시험 항목이 CANoe SIL 환경에서 실제로 실행되는 장면`

## 6. 기대효과 및 향후 계획
본 구조는 경고 컨셉 구현과 시험 운영을 하나의 자동화 기준선으로 연결하고, 반복 가능한 검증 체계를 제공한다. 향후에는 자동화 파이프라인을 통해 시험 기준 안정화와 정량 결과 확보를 단계적으로 진행하고, V2X 기반 협력인지, 신호교차로 인프라 연계(SPaT/MAP), 도로위험·군집 이벤트 공유와 같은 확장 주제로 발전시킬 계획이다.

[이미지 삽입 위치]
- 권장 이미지: 향후 계획 로드맵
- 캡션 예시: `설계 고도화, 시험 안정화, 정량 평가로 이어지는 후속 계획`
