# 개발 및 검증 환경 구축 가이드 (Virtual ECU Development)

> [!NOTE]
> 본 문서는 가상 환경(vECU)에서의 효율적인 개발 및 검증을 위한 도구와 환경 구성을 정의합니다. 고도화 작업 및 기술적 고려사항을 포함하고 있습니다.

---

## 🛠 1. 즉시 활용 가능한 전문 도구 (Vector Ecosystem)

실차 없이도 시스템 레벨의 정밀한 검증을 가능하게 하는 핵심 도구 세트입니다.

*   **vTESTstudio**:
    *   **역할**: CANoe 통합 자동화 테스트 설계 도구
    *   **특징**: 시나리오 기반의 복잡한 테스트 케이스를 그래픽하게 설계 가능. 실차 없이도 정밀한 시퀀스 검증 및 요구사항 추적성 확보.
*   **vVIRTUALtarget**:
    *   **역할**: 가상 ECU(vECU) 생성 및 실행 환경
    *   **특징**: 실제 제어기 하드웨어 없이도 AUTOSAR 스택(OS, RTE 등)을 포함한 소프트웨어 동작을 PC에서 직접 검증 가능.
*   **MATLAB/Simulink**:
    *   **역할**: 모델 기반 설계(MBD) 및 제어 로직 검증
    *   **특징**: 조명 페이드 효과, 정밀 안전 알고리즘 등을 시뮬레이션하고 검증된 로직으로부터 직접 코드를 생성하는 데 최적화.

---

## 🐍 2. 자동화 및 분석용 오픈소스 (Python-based)

테스트 자동화 및 데이터 처리를 위한 표준적인 오픈소스 라이브러리입니다.

*   **cantools**:
    *   **역할**: DBC 파일 파싱 및 메시지 처리
    *   **특징**: CAN 메시지 인코딩/디코딩 업무의 표준 도구. Python 기반 테스트 스크립트 작성 시 필수적.
*   **udsoncan**:
    *   **역할**: UDS(ISO 14229) 진단 프로토콜 구현
    *   **특징**: 가상 환경에서 OTA 및 진단 요구사항을 테스트하고 검증하기 위한 경량 라이브러리.
*   **Google Test (GTest)**:
    *   **역할**: C/C++ 유닛 테스트 프레임워크
    *   **특징**: 제어기 로직 자체의 무결성을 검증하기 위한 코드 레벨 테스트 도구.

---

## 🚀 3. 고도화 및 시각화 (Future Enhancements)

프로젝트의 시각적 완성도와 물리적 실재감을 높이기 위한 'Wow Point'용 도구입니다.

*   **3D 환경 시뮬레이터 (Unity 기반)**:
    *   **사용처**: 조명이 실제 어두운 도로에서 어떻게 비춰지는지, ADAS 카메라가 장애물을 어떻게 인식하는지 시각적 확인이 필요할 때.
    *   **효과**: 실차 대신 3D 자동차 모델을 띄우고 핸들 조작 등에 따른 조명 변화를 실시간 시각화하여 프로젝트의 생동감 부여.
*   **물리 시뮬레이션 (DYNA4)**:
    *   **사용처**: 급정거 시 차체 흔들림, 타이어 마찰력 등 물리적 거동이 안전 로직(후진 도어 개방 등)에 미치는 영향을 분석할 때.
*   **HMI 에뮬레이터 (Android/QNX)**:
    *   **사용처**: CANoe 패널을 넘어 실제 IVI 화면(안드로이드 오토모티브 등)에서의 사용자 경험을 검증할 때.
