# 멘토 피드백 반영 - 간단 버전 다이어그램

## 📋 개요
멘토 샘플의 **간단하고 개념적인 스타일**에 맞춰 재작성한 다이어그램입니다.

---

## 🎯 개선 사항

### ❌ 기존 문제점
- PlantUML이 너무 상세하고 기술적
- 컴포넌트 다이어그램이 복잡함
- 멘토가 원하는 **개념적/시각적** 접근과 차이

### ✅ 개선 방향
- **간단한 구조** 중심
- **아이콘/박스** 기반 표현
- **명확한 계층** 구조
- **최소한의 디테일**

---

## 📁 생성된 파일

### 1️⃣ **02_Concept_Design_Simple.puml**
**목적**: 시스템 개념 설계 (입력부-제어부-출력부)

**내용**:
- ✅ 3부 구조 (입력부/제어부/출력부)
- ✅ 각 부분의 역할 명시
- ✅ 데이터 흐름 표시

**멘토 샘플 매칭**:
- 입력부: 센서류 + 사용자 입력
- 제어부: Gateway + vECU
- 출력부: 화면 표시 + 액추에이터 제어


### 2️⃣ **03_Architecture_Tree_Simple.puml**
**목적**: Gateway 중심 계층 구조

**내용**:
- ✅ Central Gateway (최상위)
- ✅ 4개 Domain (중간 계층)
  - Infotainment
  - Body
  - ADAS
  - Powertrain
- ✅ 각 Domain별 ECU/센서 (최하위)

**멘토 샘플 매칭**:
- 트리 형태 구조
- CAN 버스 연결 (500kbps)
- 간단한 라인 연결


### 3️⃣ **03_System_Workflow_Simple.puml**
**목적**: vECU 중심 처리 흐름

**내용**:
- ✅ 센서 데이터 수집
- ✅ Gateway → vECU 전달
- ✅ vECU 처리 로직
- ✅ 제어 명령 생성
- ✅ 각 ECU로 전송

**추가 가치**:
- 동작 프로세스 시각화
- 긴급/일반 모드 분기
- 병렬 제어 처리

## 💡 추가 개선 제안

### 더 단순화하려면:
- **PowerPoint** 사용
  - 도형 + 아이콘으로 구성
  - 색상 구분 명확
  - 시각적 임팩트 강화

- **Draw.io** 사용
  - 드래그 앤 드롭
  - 아이콘 라이브러리 풍부
  - 간단한 편집

- 현재 간단 버전으로도 충분
- 색상 구분으로 가독성 확보
- 주석 최소화
