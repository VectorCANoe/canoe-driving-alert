## DMS 제거 결정 및 근거

> **결정자**: 준영 (팀장)
> **결정 내용**: DMS(Driver Monitoring System) 기반 눈 감김/시선 이탈 감지를 전면 제거

### DMS를 사용하면 안 되는 이유

| 문제 | 내용 |
|------|------|
| CANoe 영역 밖 | 눈 감김 감지는 OpenCV / AI 모델 영역. CANoe에서 검증 불가 |
| sysvar 시뮬의 한계 | `sysvar::DMS::EyeClosureRate = 1` 로 흉내내면 **의미 없는 개발**. 실제 눈 감김 판별 로직이 없음 |
| AI 개발 필요 | 정상 눈 깜빡임 vs 졸음성 눈 감김 구분은 PERCLOS 알고리즘 + 후처리 ML 모델 필요. 프로젝트 범위 초과 |
| 발표 공격 취약 | "눈 감김을 어떻게 판별했나요?" "정상 깜빡임과 졸음을 어떻게 구분했나요?" → 방어 불가 |
| 시중 솔루션 연동 불가 | DMS 알고리즘 엔진을 외부에서 가져와 CANoe에 연동하는 것은 현실적으로 불가능 |

### 졸음운전 감지 재정의 — 행동 기반 간접 감지

DMS 없이도 **차량 거동 신호만으로 졸음을 합리적으로 판단**할 수 있습니다.
이는 실제 완성차 메이커(Volvo, Mercedes)가 사용하는 **행동 패턴 기반 졸음 감지** 방식입니다.

```
[졸음 패턴 정의 — CANoe CAN 신호만 사용]

졸음 의심 이벤트 카운트:
  → 차선이탈(LDW) 발생           gDrowsyCount++
  → 안전거리 미확보(TTC < 3초) 발생  gDrowsyCount++
  → 급차선변경(방향지시 없이 조향) 발생  gDrowsyCount++

gDrowsyCount ≥ 3  (10분 내)
  → 졸음 패턴으로 판단 → 경고 단계 강제 상승
```

**왜 합리적인가?**
졸음 상태에서는 차선 유지 능력 저하, 차간거리 감소, 불규칙 조향이 복합 발생합니다.
개별 이벤트는 단순 부주의일 수 있지만, **복합 패턴이 누적되면 졸음 가능성 매우 높음** — 이것이 Volvo의 DAA(Driver Alert Assist) 원리입니다.

| 범주 문제 | 택천 | "실제 감속까지 이뤄진다면 경고 시스템 범주를 넘는다" | **감속 강제 실행 제거 확정**. "IVI 감속 권고 메시지 출력"으로만 한정. 실제 차속 제어는 이 프로젝트 범위 밖 |

**CAPL 구현 핵심 구조**

```
// CAPL 구간 변수 정의
int gRoadZone = 0;       // 0=일반(80), 1=스쿨존(30), 2=고속(110)
int gSpeedLimit = 80;

// Panel 구간 버튼 클릭 이벤트
on sysvar sysvar::IVI::RoadZone {
  gRoadZone = @sysvar::IVI::RoadZone;
  if (gRoadZone == 0) gSpeedLimit = 80;
  else if (gRoadZone == 1) gSpeedLimit = 30;
  else if (gRoadZone == 2) gSpeedLimit = 110;
  // → CAN 신호 자동 송출
  output(msg_SpeedLimit);
}
```

| 카운트 기준 미정 | 준영 | "10회 기준을 시간/거리 맥락 포함해서 세분화해야" | **"10분 내 3회 이상"으로 확정 권장**. CANoe 타이머 변수 1개로 구현 가능 (가장 심플). `gAccelCount` 카운터 + 10분 타이머 리셋 |
| OTA 시점 | 현준 | "주행 중 OTA 불가, P기어 후 실행 필요" | **P기어 활성 후 팝업 표시로 이미 반영 완료** ✅ (성현 수정안 채택) |
| 해제 조건 없음 | 준영 | "안전모드 해제 조건이 아직 정의되지 않음" | **Level 1**: IVI에서 운전자 수동 해제 버튼. **Level 2**: OTA 정책 활성화이므로 별도 해제 없음 (버전 유지, 재OTA로만 변경) |
| OTA 내부 흐름 | 준영 | "UDS 세션 흐름 구체화 필요" | Sample 프로젝트 OTA 흐름 그대로 재사용: `UDS 0x10(Programming) → 0x27(Security Access) → 0x34(Request Download) → 0x36×N(Transfer) → 0x37(Exit) → ECU Restart` |

**빗길 환경 모드 통합 구조**

빗길을 독립 시나리오가 아닌 **임계값 하향 조건**으로 처리:

```
BCM wiper_status = ON  →  gRainMode = 1
  → 급가속 임계값: Accel 변화율 80%/s → 60%/s 로 하향
  → 급제동 임계값: Brake 압력 80bar  → 60bar 로 하향
  → 경고 단계 상승 속도 빠르게 (10분 내 3회 → 10분 내 2회)
```

**관여 ECU**

```
입력  Accel_ECU + Vehicle_ECU + ABS_ECU + BCM(wiper)
       ↓
라우팅 CGW → WDM_ECU
       ├→ Cluster ECU   (계기판 경고)
       ├→ Ambient ECU   (조명 경고)
       ├→ IVI ECU       (팝업: "지금 활성화 / 나중에")
       └→ OTA_Server    (UDS 프로그래밍 세션)
```

**수정 필요 사항**

- [x] ECU 확장: Accel 단독 → **Accel + Vehicle + ABS + BCM(wiper) 확정**
- [x] 카운트 기준: **"10분 내 3회"로 확정**
- [x] OTA 시점: **P기어 활성 후 팝업 확정**
- [x] 해제 조건: **Level 1 수동 해제 / Level 2 버전 유지 확정**
- [ ] 빗길 임계값 하향 수치 팀 결정 필요 (60% vs 50%)

**요구사항 도출 가능성** ✅ 충분 — 급가속 감지 / 급제동 감지 / 빗길 임계값 하향 / 카운트 누적 / Level1 안전모드 / Level2 OTA

### Scenario C — 차선이탈 + 급차선변경 + 전방주시태만/졸음 `담당: 현준 (+ 라엘?)`

> 차선 이탈·급차선변경·전방주시태만·졸음운전 → 다단계 다감각 경고 → Safe State 진입
> 위험행동 ②④⑦⑧ 통합 커버

**팀 피드백 쟁점 및 솔루션**

| 쟁점 | 제기자 | 내용 | 솔루션 |
|------|:------:|------|--------|
| 두 기능 구분 모호 | 준영 | "시선 이탈과 차선 이탈을 묶으면 구분이 모호해짐" | **역할 분리 확정**: 차선이탈(LDW) + 급차선변경(MDPS 조향각)을 메인 트리거로 사용. 시선이탈·졸음운전(DMS)은 **완전 제거** → `gDrowsyCount` 행동 패턴 카운터로 대체. 카메라·AI 없이 CANoe CAN 신호만으로 구현 |
| 카메라 외부 의존 | 준영 | "외부 센서 의존도 높음 → 내부 변환 방법 모색 필요" | **카메라 완전 불필요**. `sysvar::LDW::LaneDeviation` (차선이탈), MDPS_ECU 조향각 (급차선변경), Front_Radar TTC (안전거리) → CANoe Panel에서 직접 주입. ~~DMS sysvar 완전 제거~~. = SIL(Software-In-the-Loop) 표준 방식 |
| 터치 제한 범위 | 택천 | "주행 관련 기능 터치가 제한되면 안 됨" | **"비주행 기능만 제한"으로 확정**: 음악앱·스트리밍·SNS 제한. 내비게이션·비상연락·경고 해제 버튼은 항상 활성 유지 |
| ECU 다양성 부족 | 현준 | "다양한 ECU와 연동되면 좋겠음" | 급차선변경(MDPS_ECU) + 차선이탈(LDW_ECU) + 안전거리(Front_Radar) 3개 입력 신호 사용. ~~DMS_ECU 완전 제거~~ → `gDrowsyCount` 카운터로 졸음 판단. 출력 7개 확장 |

**트리거 구조 정의 (행동 기반 — DMS 없음)**

| 트리거 | 신호 | 조건 |
|--------|------|------|
| 차선이탈 | `sysvar::LDW::LaneDeviation` | 편차 발생 시 `gDrowsyCount++` |
| 급차선변경 | MDPS_ECU 조향각 변화율 | Δθ/Δt > 임계값 + 방향지시등 OFF → `gDrowsyCount++` |
| 안전거리 미확보 | Front_Radar TTC | TTC < 3초 → `gDrowsyCount++` |
| **졸음 패턴 판단** | `gDrowsyCount` 누적 카운터 | **≥ 3회 / 10분 내 → 졸음 경고 강제 발동** |

→ **차선이탈·급차선변경·안전거리 이벤트가 누적되어 `gDrowsyCount ≥ 3` 달성 시 졸음 단계 직행. 개별 이벤트도 즉시 1단계 경고 진입**

**관여 ECU**

```
입력  MDPS_ECU + LDW_ECU + Front_Radar_ECU (행동 기반, DMS 없음)
       ↓
라우팅 CGW → WDM_ECU (gDrowsyCount 집계)
       ├→ Cluster ECU  (경고 텍스트)
       ├→ HUD ECU      ("핸들을 잡아주세요")
       ├→ Seat ECU     (시트 진동, 1단계~)
       ├→ Sound ECU    (경고음, 2단계~)
       ├→ IVI ECU      (화면 Dim + 비주행 터치 제한)
       ├→ MDPS ECU     (스티어링 햅틱 진동, 2단계~)
       └→ ACC ECU      (30km/h 감속, 3단계 Safe State)
```

**수정 필요 사항**

- [x] 차선이탈 + 급차선변경 메인 트리거 구조 **확정**
- [x] 터치 제한: **"비주행 기능만 제한" 확정**
- [x] 카메라 → **System Variable + Panel 구현 확정**
- [x] ~~DMS_ECU~~ 완전 제거 → **`gDrowsyCount` 행동 패턴 카운터로 대체 확정**
- [x] 졸음 판단: **차선이탈 + 안전거리 미확보 + 급차선변경 ≥ 3회 / 10분 = 졸음으로 간주 확정**

**요구사항 도출 가능성** ✅ 충분 — 트리거 3종 정의 / `gDrowsyCount` 누적 로직 / 1단계 경고(HUD) / 2단계 다감각 / 3단계 Safe State / 속도 차등 / 복귀 조건

### Scenario D — 차간거리 미확보 + BSD 후방 충돌 도어 잠금 `담당: 라엘`

> 전방 차간거리 미확보(전방 TTC) + 후방 충돌 위험(BSD TTC) → 다채널 경고 + 도어 잠금
> 위험행동 ③ 전방/후방 통합 커버

**팀 피드백 쟁점 및 솔루션**

| 쟁점 | 제기자 | 내용 | 솔루션 |
|------|:------:|------|--------|
| 직접 피드백 없음 | 전체 | 팀 의견 모음에서 아직 본격 검토 안 됨 | 피드백이 없는 이유는 검토가 안 됐기 때문. 실제로는 **4개 시나리오 중 조건·흐름·해제 조건이 가장 명확**. 전방 TTC 추가로 REQ 6개 완성 |
| 관심 표명 | 라엘 | "후방 안전 / 주차에도 관심 있다" | **라엘 담당으로 확정 권장**. 후방(BSD) + 전방(Front Radar) 통합으로 차간거리 미확보 전 방향 커버 |

**전방 + 후방 통합 구조**

| 구분 | 전방 (차간거리 미확보) | 후방 (BSD 충돌 위험) |
|------|:-------------------:|:-----------------:|
| 입력 ECU | Front_Radar_ECU | BSD_ECU |
| 조건 | TTC_Front < 3초 | TTC_Rear < 2초 |
| 주요 출력 | Cluster + HUD + Sound | Door 잠금 + Mirror LED + Ambient |
| 활성 차속 | 주행 중 (모든 속도) | 저속/정차 (≤ 10km/h) |

**관여 ECU**

```
입력  Front_Radar_ECU + BSD_ECU
       ↓
라우팅 CGW → WDM_ECU
       ├→ Cluster ECU   (경고 텍스트)
       ├→ HUD ECU       (전방 경고 표시)
       ├→ Sound_ECU     (경고 비프음)
       ├→ Door_ECU      (후방 위험 시 잠금 지연 3초)
       ├→ Mirror_ECU    (사이드미러 LED 점멸)
       └→ Ambient_ECU   (빨간색 전환)
```

**요구사항 도출 가능성** ✅ 충분 — 전방 TTC 감지 / 전방 경고 단계 / 후방 BSD 감지 / 도어 잠금 지연 / 방향별 LED / 자동 해제
