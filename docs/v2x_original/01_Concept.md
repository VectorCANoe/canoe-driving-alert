# 특수목적 차량 V2V 통신 검증 플랫폼

**Date**: 2026-02-24
**Status**: Concept Draft

---

## 한 줄 요약

> "자동차들이 서로 직접 대화해서, 경찰차는 길을 열고, 군용차는 일사불란하게 움직이고, 물류차는 재난 순간 긴급차량으로 변신한다."

---

## 왜 이 주제인가

지금까지 차량 기술 연구는 대부분 **한 대의 차량 안에서** 일어나는 일에 집중했습니다.
이 프로젝트는 한 단계 위입니다. **차량과 차량이 직접 통신하면(V2V) 어떤 일이 가능한지** 검증합니다.

경찰차가 긴급 출동할 때 사람이 직접 경적을 울리거나 수신호를 보내는 대신,
주변 차량들이 **자동으로 경고를 받고 길을 비킵니다**.

군용차 호송 중 선두 차량이 위협을 감지하면,
뒤따르는 차량들이 **0.1초 안에 동시에 같은 행동을 합니다**.

평시에 물건을 나르던 트럭이 재난이 선포되는 순간,
국가 서버에서 무선으로 업데이트를 받아 **긴급차량으로 바뀝니다**.

---

## 전장(戰場) 통신 환경의 변화와 연구 정당성

### 기존 한계 — 지상 기지국 의존

과거 전장에서 군용 차량들은 **지상 군용 통신망(JTRS, VHF/UHF 무전)** 에 의존했습니다.
이 방식의 치명적 약점은 명확합니다.

```
기지국 또는 중계기가 공격·파괴되는 순간
        ↓
  전장 전체의 외부 통신이 단절된다
        ↓
  차량 간 명령 전달 불가 → 군집 작전 붕괴
```

이 때문에 군집 차량 간 실시간 협력 통신은 **항상 제한적이고 불안정한 환경** 에 놓여 있었습니다.

---

### 현재 변화 — 위성 통신의 전장 도입

스타링크(Starlink) 등 저궤도 위성 통신 기술의 등장으로 상황이 근본적으로 바뀌었습니다.
우크라이나 전쟁에서 실전 검증되었듯, **지상 기지국 없이도 전장 어디서나 안정적인 광대역 통신이 가능** 합니다.

| 구분 | 기존 지상 통신망 | 위성 통신 (스타링크 등) |
|------|----------------|----------------------|
| 통신 유지 조건 | 기지국/중계기 생존 필요 | 위성 위성망 유지 (지상 거점 불필요) |
| 기지국 공격 시 | 통신 단절 | 영향 없음 |
| 전장 커버리지 | 기지국 반경 제한 | 지구 전역 |
| 차량 간 직접 통신 | 별도 인프라 필요 | V2V + 위성 백홀 동시 가능 |

이제 **차량 간 직접 통신(V2V)** 은 더 이상 평시 도로 환경만의 기술이 아닙니다.
전장에서도 위성 통신을 백홀(backhaul)로 삼아 차량 노드 간 실시간 협력이 가능한 시대가 열렸습니다.

---

### 이 프로젝트의 의미

이 프로젝트는 **전장 통신 인프라가 갖춰진 환경을 가정**하고,
차량 노드들이 외부 의존 없이 Ethernet V2V로 직접 대화하는 아키텍처를 검증합니다.

> 위성 통신이 확보된 전장에서, 군집 차량이 외부 명령 대기 없이 자체적으로 즉각 반응하는 구조 —
> 이것이 이 연구가 다루는 핵심 시나리오입니다.

---

## 3가지 시나리오

### 시나리오 1 — 긴급 통행권 (Emergency Corridor)

```
경찰차가 긴급 출동 버튼을 누른다
         ↓
   Ethernet으로 주변 차량 전체에 신호 전송
         ↓
일반차 A, B, C의 계기판에 동시에 팝업이 뜬다
   "긴급 차량 접근 중 — 우측 양보 바랍니다"
         ↓
경찰차가 지나간 후 긴급 모드 해제 → 경고 자동 소거
```

**핵심 기술**: Ethernet 브로드캐스트 (1대가 여러 대에게 동시 전달)

---

### 시나리오 2 — 군집 위협 대응 (Platoon Defense)

```
선두 군용차가 위협을 감지한다 (드론 공격 / 지뢰 탐지)
         ↓
   Ethernet으로 후행 차량 전체에 명령 전송
         ↓
   Level 1 (경계) → 모든 후행차 경계등 점멸
   Level 2 (회피) → 모든 후행차 동시 비상 제동
   Level 3 (스텔스) → 모든 후행차 전 조명 즉시 소등
         ↓
   위협 해제 명령 → 전체 복귀
```

**핵심 기술**: Ethernet 동기 명령 (선두차 명령 → 후행차 전체 동시 실행)

---

### 시나리오 3 — 물류차 → 긴급차 임무 전환 (Mission Change OTA)

```
재난 상황 발생 → 국가 명령 서버에서 OTA 명령 전송
         ↓
   물류차가 Ethernet으로 "긴급차량 모드" 펌웨어를 다운로드
         ↓
   무결성 검증(CRC) 통과 → 차량 재시작
         ↓
   사이렌 ON + 앰비언트 적청 교차 + 속도 제한 해제
   계기판: "긴급차량 모드 활성"
```

**핵심 기술**: Ethernet OTA (서버 → 차량 펌웨어 배포 + 검증 + 복구)

---

## 통신 구조 및 설계 타당성

본 프로젝트는 물리적인 무선 안테나 대신 **Ethernet(UDP)과 CAN 네트워크 하이브리드 구성**을 채택하여 V2V 통신을 시뮬레이션합니다. 이는 현업 선행연구(Advanced Research)의 아키텍처 검증(PoC) 방법론을 따른 효율적인 설계입니다.

### 아키텍처 채택 배경

1.  **상위 애플리케이션 계층(L7) 로직 검증에 집중**: 이 프로젝트의 핵심은 물리 계층(PHY)의 통신 지연 측정이 아니라, **분산된 다중 차량 노드 간의 제어 동기화(Control Sequence)**를 구현하는 것입니다. Ethernet 환경은 네트워크 변수를 통제하여 차량 간의 상태 유발(Trigger)과 물리적 반응(Actuation) 로직 자체의 정합성을 최우선으로 검증할 수 있게 합니다.
2.  **현업 선행연구 표준 프로세스 적용**: 실제 완성차 R&D에서도 초기 로직 설계 단계부터 고가의 V2X 칩셋(OBU/RSU)을 사용하지 않습니다. **이더넷 기반의 HILS/SILS 환경을 구축해 제어 정합성부터 선행 검증**하는 것이 차량 소프트웨어 개발의 합리적인 접근법입니다.
3.  **강력한 시스템 이식성(Portability) 확보**: 외부 네트워크(Ethernet) 메시지를 수신하여 내부망(CAN)으로 라우팅하는 게이트웨이 아키텍처는 통신 매체와 무관하게 동일하게 동작합니다. 현재 검증된 코드와 제어 로직은 향후 물리적인 무선 V2X 모듈만 결합하면 즉시 실차 환경으로 포팅될 수 있는 유연한 구조를 지닙니다.

### 통신 데이터 흐름

| 통신 | 역할 | 사용 구간 |
|------|------|---------|
| **Ethernet** | 차량 ↔ 차량 (V2V) 통신 모사 | 경찰차↔일반차, 선두차↔후행차, 서버↔물류차 |
| **CAN** | 실제 차량 내부 ECU 제어망 | Ethernet 명령 수신 후 → 계기판/앰비언트/사이렌 등 하드웨어 액추에이터 구동 |

**동작 원리**: Ethernet으로 차량 간 논리적 대화(Trigger 송수신) → CAN으로 차량 내부 물리적 실행(Actuation)

```text
[차량 A (송신)] ──── Ethernet UDP ────→ [차량 B (수신)]
  (상황 판단)                                ↓
                                 [차량 B 내부 CAN 버스 라우팅]
                                            ↓
                                계기판 / 앰비언트 / 사이렌 제어
```

## 차량 노드 구성

| 노드 이름 | 역할 | 통신 |
|---------|------|------|
| **Police_Node** | 긴급 출동 트리거 → 브로드캐스트 발송 | Ethernet 송신 |
| **Civ_Node A/B/C** | 긴급 메시지 수신 → 계기판/앰비언트 경고 | Ethernet 수신 → CAN |
| **Lead_Node** | 위협 감지 → 군집 명령 발송 | Ethernet 송신 |
| **Follow_Node 1/2/3** | 군집 명령 수신 → 동시 실행 | Ethernet 수신 → CAN |
| **Server_Node** | 국가 OTA 서버, 펌웨어 배포 | Ethernet 송신 |
| **Logistics_Node** | 물류차, OTA 수신 → 임무 전환 | Ethernet 수신 → CAN |

## 기존 승용차 프로젝트와의 차이

| 항목 | 기존 (SDV 경험 플랫폼) | 이 프로젝트 (V2V) |
|------|-------------------|--------------------|
| 대상 | 일반 승용차 1대 | 특수목적 차량 여러 대 |
| 통신 | 차량 내부 CAN | 차량 간 Ethernet + 내부 CAN |
| 주제 | 편의/UX 개선 | 공공 안전 / 군사 임무 수행 |
| 차별성 | 구현 깊이 | 주제 + 멀티노드 통신 |
