# 포스터 세션 초안

## 제목
구간 정보와 긴급차량 접근을 통합한 실시간 주행 경고 컨셉  
부제: CANoe SIL 기반 구현, 시험 구조, 검증 운영 자동화

## 1. 문제 정의
주행 맥락과 긴급 상황을 함께 반영하지 못하면 경고가 개별 기능 단위로 분산된다. 이 경우 동일 시점에 여러 경고가 겹치면서 우선순위 충돌, 표시 중복, 복귀 불안정이 발생할 수 있다. 따라서 서로 다른 입력을 하나의 기준으로 해석하고 단일 경고로 중재하는 구조가 필요하다.

[이미지 삽입 위치]
- 도심 주행, 스쿨존, 긴급차량 접근, 전방 위험이 동시에 겹치는 문제 상황 개념도

## 2. 프로젝트 목표
본 프로젝트는 현대 차량 시스템을 참고한 기본 ECU 구성 위에서 주행 경고 컨셉을 다시 구성하는 것을 목표로 하였다. 구간 정보, 긴급차량 접근, 객체 위험을 하나의 경고 정책으로 연결하고, 이를 CANoe SIL에서 구현 및 시험 가능한 구조로 구체화하였다. 구현과 시험 범위는 라이선스와 검증 조건을 고려하여 CAN과 Ethernet으로 한정하였다.

[이미지 삽입 위치]
- 프로젝트 범위: Hyundai-style ECU baseline, CAN/Ethernet, CANoe SIL, CAPL, UT/IT/ST

## 3. 시스템 구조
시스템은 차량 상태 수집, 긴급 이벤트 정규화, 위험도 산정 및 경고 중재, HMI 및 감속 보조 출력의 네 단계로 구성하였다. CAN은 차량 상태와 구간 맥락 수집에 사용하고, Ethernet은 긴급 이벤트 전달과 경고 연계에 사용하였다. 또한 운영 경로와 검증 경로를 분리하여 경고 동작과 시험 관측이 동시에 가능하도록 구성하였다.

[이미지 삽입 위치]
- ECU 블록도
- CAN/Ethernet 혼합 네트워크 흐름도

## 4. 핵심 기능
V2X는 긴급차량 유형, ETA, SourceID를 기준으로 활성 이벤트를 정규화하고 timeout clear를 관리한다. ADAS는 긴급 접근 정보와 객체 위험 정보를 통합하여 위험도를 산정하고, 계층형 경고 중재와 감속 보조 요청을 수행한다. 이 과정에서 hysteresis, stale 처리, fail-safe를 함께 적용하여 경고 선택과 복귀가 과도하게 흔들리지 않도록 하였다.

[이미지 삽입 위치]
- V2X 이벤트 선택 상태도
- ADAS 위험도 산정 및 경고 중재 블록도

## 5. 구현 및 구축 성과
본 프로젝트는 CAPL 기반 논리 ECU, Validation Harness, CANoe SIL 실행 환경을 통해 경고 컨셉을 실제로 실행 가능한 구조로 구현하였다. 또한 요구사항부터 기능, 통신, 변수, 코드, UT/IT/ST까지 연결되는 추적 구조를 정리하고, 이를 운영하기 위한 자동화 파이프라인 콘솔을 함께 구성하였다. 콘솔은 `verify batch`, `Execution Manifest`, `run insight`, `doc binding bundle`, `doc fill template` 등 closeout 중심 운영 흐름을 제공하며, 현재 기준선에서 `cfg-hygiene`, `cli-readiness`, `text-integrity` 게이트 통과를 확인하였다.

대표 시험 항목도 실행 가능한 형태로 고정하였다. 예를 들면 `UT_004`, `UT_010`은 긴급 이벤트 수신과 `1000ms` timeout clear를 다루고, `IT_004`, `IT_005`, `IT_006`은 경찰·구급 수신 및 우선순위 중재를, `ST_016`, `ST_017`, `ST_018`, `ST_019`, `ST_020`, `ST_021`은 ETA·SourceID 기반 선택, `100ms` 외부 송신, timeout clear 이후 복귀를 대표한다. 현재 단계에서는 정량 성능 수치보다 핵심 경고 컨셉과 시험 구조를 실제 실행 가능한 형태로 연결한 점에 의미가 있다.

[이미지 삽입 위치]
- CANoe 실행 화면
  - 캡션 예시: `CANoe Test Unit 실행 화면 예시`
  - 보조 문구: `대표 시험 항목이 CANoe SIL 환경에서 실제로 실행되는 장면으로, 구현 자산과 시험 구조가 연결된 상태를 보여준다.`
- Validation Harness/trace 화면
- 운영 콘솔 TUI 또는 결과 산출물 예시

## 6. 기대효과 및 향후 계획
본 구조는 혼합 CAN/Ethernet 기반 경고 컨셉을 설명하고 반복 시험과 closeout 운영을 수행할 수 있는 기준선으로 활용될 수 있다. 향후에는 V2X-ADAS-CGW 경계 정제, 시험 기준 안정화, 정량 결과 확보, 대표 시나리오별 근거 데이터 축적을 통해 검증 수준을 단계적으로 높일 계획이다. 이를 통해 기능 문제와 구조 문제를 구분하면서 신기능 컨셉의 적용 가능성을 점진적으로 높일 수 있다.

[이미지 삽입 위치]
- 향후 계획 로드맵
- 설계 고도화 -> 시험 안정화 -> 정량 평가 단계도
